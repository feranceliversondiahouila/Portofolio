AC25.02Cyber : Prendre en main des outils de test de pénétration réseau/système

Utiliser les outils offensifs de référence (dans un cadre légal et éthique) pour tester la robustesse d'une infrastructure réseau et système.

📚 Modules associés

Les ressources (modules) qui travaillent cette compétence :

  • R504 — Pentest réseau — Nmap, Metasploit, Wireshark en contexte offensif
  • R505 — Pentest système — élévation de privilèges, post-exploitation
  • R506 — Environnements de lab — Kali Linux, machines vulnérables (HackTheBox, TryHackMe)

🧠 Notions clés à maîtriser

  • Kali Linux : distribution dédiée au pentest — outils principaux
  • Méthodologie pentest : Reconnaissance → Scan → Exploitation → Post-exploitation → Rapport
  • Nmap : scan de ports, détection de services, scripts NSE
  • Metasploit Framework : modules exploit, payload, meterpreter
  • Wireshark en contexte offensif : capture de credentials en clair
  • Hydra / Medusa : attaque par dictionnaire sur SSH, FTP, HTTP
  • Nikto / dirb / gobuster : scan de vulnérabilités web
  • Élévation de privilèges Linux : SUID, sudo -l, cron jobs
  • Windows privilege escalation : tokens, services, GPO mal configurés
  • Rédaction d'un rapport de pentest : executive summary, findings, CVSS, remédiation
  • CTF (Capture The Flag) : HackTheBox, TryHackMe, Root-Me

🛠️ SAÉ liées

Situations d'apprentissage et d'évaluation mobilisant cette compétence :

  • SAE51 — Pentest d'une infrastructure virtuelle en environnement contrôlé
  • SAE52 — Participation à un CTF et rédaction d'un writeup
