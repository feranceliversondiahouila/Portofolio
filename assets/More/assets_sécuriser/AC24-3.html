AC24.03Cyber : Sécuriser les services

Durcir et sécuriser les services exposés (web, mail, DNS, SSH) en appliquant les bonnes pratiques de configuration et de hardening.

📚 Modules associés

Les ressources (modules) qui travaillent cette compétence :

  • R407 — Hardening des services — SSH, Apache/Nginx, DNS, SMTP
  • R408 — Gestion des certificats — PKI, Let's Encrypt, HTTPS
  • R409 — Authentification et contrôle d'accès — IAM, MFA, LDAP

🧠 Notions clés à maîtriser

  • SSH : désactivation root, clés publiques/privées, fail2ban
  • HTTPS : configuration TLS 1.3, HSTS, certificats Let's Encrypt
  • Hardening Apache/Nginx : headers de sécurité, modules désactivés
  • DNS sécurisé : DNSSEC, DNS over HTTPS (DoH)
  • SMTP : SPF, DKIM, DMARC — lutte contre le spoofing
  • Authentification forte : MFA (TOTP, U2F)
  • LDAP / Active Directory : contrôle d'accès basé sur les rôles (RBAC)
  • Principe du moindre privilège

🛠️ SAÉ liées

Situations d'apprentissage et d'évaluation mobilisant cette compétence :

  • SAE41 — Hardening d'un serveur Linux exposé sur Internet
  • SAE42 — Mise en place d'une PKI interne et de certificats TLS
